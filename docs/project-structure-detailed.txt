Project Architecture and Structure (Updated 2026-03-04)
======================================================

Overview
--------
Code to Prompt Generator is a hybrid app:
- Frontend: Next.js + React + TypeScript
- Desktop shell: Electron
- Backend: Flask (Python)

Runtime defaults:
- Frontend dev server: 3010
- Backend API: 5010

Top-Level Directories
---------------------
- `pages/`: Next.js routes. `pages/index.tsx` is the main application screen.
- `views/`: Feature-level UI sections (file tree, prompts, kanban, user stories, settings, etc.).
- `views/layout/`: Main page composition (`HeaderView`, `MainLayoutView`, `LeftPanelView`, `RightPanelView`).
- `components/ui/`: Reusable UI primitives (buttons, dialogs, tabs, etc.).
- `stores/`: Zustand stores for app/project/prompt/exclusions/todos/selection groups.
- `services/`: Frontend API hooks and integration logic.
- `lib/`: Shared frontend utilities and composition hooks.
- `python_backend/`: Flask API, services, repositories, and tests.
- `electron/`: Electron main-process logic.
- `scripts/`: Development and automation scripts (`postinstall`, `autotest`, backend runner).
- `docs/`: Technical documentation.

Frontend Flow
-------------
1. `pages/index.tsx` uses `useRefactoredHomePageLogic`.
2. `useRefactoredHomePageLogic` composes smaller hooks:
   - `useDataInitialization`
   - `useProjectLogic`
   - `useUIState`
   - `useServiceActions`
3. Layout is rendered by `MainLayoutView`:
   - Left side: file tree, selection groups, exclusions, and task tabs.
   - Right side: prompt editor/output controls.
4. State is centralized in Zustand stores; services call backend via `services/apiService.ts`.

Selection Groups (Current Design)
---------------------------------
- Source of truth: backend file per project (`.codetoprompt/selection_groups.json`).
- Frontend store: in-memory cache only (`stores/useSelectionGroupStore.ts`).
- API integration: `services/selectionGroupServiceHooks.ts`.
- Migration: first load can migrate legacy localStorage groups (`ctpgSelectionGroups_v1`) into backend.

Backend Layout
--------------
- `python_backend/app.py`: Flask app bootstrap, CORS, blueprint registration, health endpoint.
- `python_backend/controllers/`: HTTP endpoints.
- `python_backend/services/`: business logic (project tree, todos, codemap, prompt refine, etc.).
- `python_backend/repositories/`: file-based persistence helpers.
- `python_backend/models/`: request validation schemas.
- `python_backend/utils/`: response helpers, path utilities.
- `python_backend/tests/`: backend endpoint/service tests.

Main API Groups
---------------
- Health: `/health`
- Projects: `/api/projects/tree`, `/api/projects/files`
- Exclusions: `/api/exclusions`, `/api/localExclusions`
- Todos: `/api/todos`
- Selection Groups: `/api/selectionGroups`
- Kanban: `/api/kanban`
- User Stories: `/api/user-stories`
- Prompt tools: `/api/prompt/refine`, `/api/autoselect`, `/api/codemap/extract`, `/api/tokenCount`
- Folder utils: `/api/drives`, `/api/select_drives`, `/api/browse_folders`, `/api/resolveFolder`
- Meta prompts: `/api/metaprompts`

Quality Gates
-------------
- Lint: `npm run lint`
- Typecheck: `npm run typecheck`
- Frontend build: `npm run build`
- Backend tests: `python_backend/venv/bin/pytest -q python_backend/tests`
- Integration checks: `npm test`

CI (`.github/workflows/ci.yml`) runs backend tests, frontend quality gates, and integration autotests.
